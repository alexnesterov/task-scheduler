# task-scheduler

HTTP API для управления задачами с поддержкой правил повторения.
Написан на Go, хранилище — SQLite.

## Требования

- Go 1.25+

## Запуск

```bash
go run ./cmd/app/main.go
```

Сервер запускается на порту `7540` и создаёт `scheduler.db` при первом
запуске.

## Конфигурация

Конфиг читается из `configs/config.yaml`. Любое значение можно
переопределить через переменную окружения с префиксом `TSCHED_`:

```bash
TSCHED_HTTP_PORT=8080 TSCHED_DB_FILE=prod.db \
  go run ./cmd/app/main.go
```

## API

| Метод | Маршрут | Описание |
| ------ | ------- | -------- |
| GET | `/api/nextdate?now=&date=&repeat=` | Следующая дата |
| POST | `/api/task` | Создать задачу |
| GET | `/api/tasks` | Список задач (до 50) |
| GET | `/api/task?id=` | Получить задачу по ID |
| PUT | `/api/task` | Обновить задачу |
| DELETE | `/api/task?id=` | Удалить задачу |
| POST | `/api/task/done?id=` | Отметить как выполненную |

### Поля задачи

```json
{
  "id":      "1",
  "date":    "20260610",
  "title":   "Купить продукты",
  "comment": "",
  "repeat":  "d 7"
}
```

- `date` — формат `YYYYMMDD`. Если пусто, подставляется сегодня.
- `repeat` — `d <N>` (каждые N дней, N ≤ 400) или `y` (ежегодно).
  Пустое значение — одноразовая задача.

### Поведение «выполнено»

Одноразовая задача удаляется. Повторяющаяся — получает следующую дату
по правилу повторения.
